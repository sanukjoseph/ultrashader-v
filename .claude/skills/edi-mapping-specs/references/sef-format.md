# SEF format reference

This note summarizes how to read the SEF guides in this repository.

- The SEF files describe X12 transaction structure, loops, segments, elements, codes, and cardinality.
- Read the relevant loop/segment/element entry directly rather than the whole file.
- Use the requirement designators and masks shown in the guide to understand whether a segment or element is mandatory, optional, conditional, repeating, or nested.
- For code lists and value translation, also consult the related crosswalk files in Edifecs/Crosswalk.
